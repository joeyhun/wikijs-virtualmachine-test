Login to balfa and open the System Center Configuration Manager console

Click on Monitoring -> System Status -> Site Status and ensure all systems have a green checkmark next to them.

Click on Monitoring -> System Status -> Component Status and ensure all systems have a green checkmark next to them

Click on Software Library -> Software Updates -> All Software Updates and sort the updates by date released. You should have updates from the last patch tuesday available (exclude definition updates).

Finally, go to Software Library -> Software Updates -> Software Update Groups and ensure that update groups have been deployed at the latest patch tuesday (look at the deployed field and last date modified or date created field).

Verify Workstations are receiving updates properly

Go to Monitoring -> Deployments and double click on the most recent software update deployment group.

In the Asset Details pane, you can right click on a computer and select details to see which updates are having issues.

Review any errors to ensure updates are being applied correctly. (In this case, one 3rd party update is failing across all computers)

Applying new 3rd party updates

Launch System Center Updates Publisher on Balfa

Click on Import

Select the catalogs that need to be downloaded

Click Next and Next

Click Yes or Yes to All when prompted and then hit close when finished

Navigate to the software folder

Expire any old updates manually, by selecting the update and then clicking Expire.

After an update is expired, you must re-publish the expired update and the most recent update so that SCCM knows that the update is expired.

Review the Name,Expired and Date Published fields to appropriately make changes to the catalog.

After all the 3rd party updates have been published, you can wait until SCCM deploys the updates on the day after patch tuesday.
