Add Server Farm

To add a reverse proxy rule for a backend webserver that does NOT need authentication, login to volagi and villiger and pull up IIS manager.

Right click on server farms and click create server farm

Enter a friendly name for the server farm and click next

Type in the server address of the backend webserver and click Add.

Note: if you want to change which port to proxy to, click on Advanced settings, expand applicationRequestRouting and change the port for httpPort and httpsPort.

Important: After completing the add farm wizard, a popup will ask you if you want to create a rule, click NO

Create the server farm on both servers (configurations do not synchronize)

Add URL Rewrite Rule

Open URL Rewrite Module and click "Add Rule(s)..." on the right hand side.

Select "Blank rule" and click ok

The Match URL conditions will be specific to each case, but always set a condition to match the domain name, otherwise the rule will apply to any website.

Finally, ensure you route the request to the appropriate server farm and then select stop processing of subsequent rules
