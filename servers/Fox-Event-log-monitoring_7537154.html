All new servers that should have windows event log monitoring need to be added to a windows group called "Server-Auditing" located in Groups-Users OU. Once the computer object is a member of the server-auditing group, it will have the permissions needed to read and apply the "Server Auditing" GPO in active directory that is published in the servers and domain controllers OU.

Besides what events to monitor, specified in the GPO, the GPO also instructs the servers to point to fox for event subscription details.

The event subscription in Fox is currently set to subscribe to all windows events in the local Security and System event log.

After the windows event logs aggregate onto the Fox server, I configured the windows event log "Forwarded Events", where the events get dumped, to be located on the D Drive, which is stripped/mirrored on the 15k rpm disks.

After 24 hours, the logs get moved over to C:\Windows_Archive\, of which, they stay for 3 days before being deleted.

Per day, we store about 43 GB of windows event logs. In the event of a server compromise, these events will be invaluable
