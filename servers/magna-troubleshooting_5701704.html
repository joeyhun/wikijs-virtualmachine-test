Disk usage problem: sharepoint web startup script leaves temporary files that build up and take up disk space.

Delete all files in C:\Windows\System32\config\systemprofile\AppData\Local\Microsoft\Windows\INetCache\IE\ and C:\Windows\SysWow64\config\systemprofile\AppData\Local\Microsoft\Windows\INetCache\IE\

In some cases, I've had to use command prompt "DEL /F *" within the directory to delete the files, because explorer was unresponsive when trying to delete the files.

Important: To reduce interruption to the user when performing NLB failover, ensure that you run SPBestWarmUp task in the Task Scheduler prior to making the host live..this will load the web applications before hand. Otherwise the end user will have to wait 30+ seconds as the web apps load.
