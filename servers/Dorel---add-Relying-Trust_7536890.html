To add a relying trust/SSO, login to Dorel (Primary ADFS node)

Open the AD FS console

Expand "Trust Relationships" and then expand "Relying Party Trusts"

Click on "Add Relying Party Trust.." on the right hand window titled Actions

Note: Non-Claims-Aware Relying Party refers to the use of kerberos as an authentication, otherwise SAML is used.

After clicking on add relying party trust, continue through the wizard.

Add the URL to the federation metadata

Continue until the end of the wizard and allow the "Open the Edit Claim Rules dialog.." to be checked.

Click on Add Rule for Issuance Transform Rules

Select "Send LDAP Attributes as Claims"

In the next screen, select Active Directory as the attribute store and provide a mapping of any attributes you want to send to the relying party

Troubleshooting

Any problems should show up in the windows event log on peugeot and dorel under "Applications and Services Logs" -> AD FS -> Admin
