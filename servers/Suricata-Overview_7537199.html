Suricata is installed to /usr/local/etc/suricata/

Suricata logs are located at /usr/local/var/log/suricata/

Crontab is set to download new IDS signatures from emerging threats pro at 7:00am every day (launches /root/update-suricata)

#!/bin/bash

/usr/local/bin/oinkmaster.pl -o /usr/local/etc/suricata/rules/ > /usr/local/scripts/ids_refresh.txt 2>&1

cat /usr/local/etc/suricata/rules/sid-custom.map >> /usr/local/etc/suricata/rules/sid-msg.map

systemctl restart suricata > /dev/null

systemctl restart barnyard2 > /dev/null

Oinkmaster is used to pull new signatures (its configuration is at /etc/oinkmaster.conf)

When suricata alerts to an event, it logs the event in unified format to /usr/local/var/log/suricata/unified2.alert, which barnyard2 converts and sends to syslog (retaining the packet data), rsyslog then saves the log to /var/log/intrusion.log. Splunk universal agent then reads this file and sends it to splunk in the intrusion index.
